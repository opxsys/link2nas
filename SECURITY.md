# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in **Link2NAS**, please do **not** open a public GitHub issue.

Instead, report it responsibly by contacting the maintainer directly.

### How to report
- Describe the vulnerability clearly
- Include steps to reproduce if possible
- Indicate the affected component(s)
- Provide logs or screenshots if relevant (sanitize secrets)

### Scope
This project handles:
- API keys (AllDebrid)
- Credentials (Basic Auth, Synology DSM)
- Network-exposed services

Security issues are taken seriously and will be addressed as quickly as possible.

Thank you for helping keep Link2NAS secure.
